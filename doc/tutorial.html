Introduction

This tutorial will show you on hand of a real world example how you design, setup and implement an application with OOS. You will learn

  • how to design and create your classes
  • how to create object relations
  • how to setup the object store
  • how to setup a database connection
  • how to deal with database transactions

Goal

Goal of this project is to create a simple music database. With the resulting applicaction you'll be able to setup a little music database. The program will implement several commands which will work on the database.

Schema Design

We will have three main object types: Artist, Album and Track. Furthermore the artist needs a list of albums and each album needs am array of tracks. So that we defined our types we can fill 'em with their attributes. This will lead us to the following table layout:

artist
Name Type Description
PK id INTEGER primary key
name VARCHAR(64) name of artist
track
Name Type Description
PK id INTEGER primary key
title VARCHAR(64) title of track
FK artist INTEGER foreign key of artist
duration INTEGER length of track
album
Name Type Description
PK id INTEGER primary key
name VARCHAR(64) name of album
FK artist INTEGER foreign key of artist
year INTEGER year of release

These are the main object types. Now we need to helper object types. The first carries the track to album releation and the second arries the album to artist releation. Because we will use a list and a vector in the artist and album objects these object types will be auto-generated. But we will come to this later. First the layout:

track_album_rel
Name Type Description
PK id INTEGER primary key
FK value INTEGER foreign key to track
FK container INTEGER foreign key to album
year INTEGER year of release
album_artist_rel
Name Type Description
PK id INTEGER primary key
FK value INTEGER foreign key to album
FK container INTEGER foreign key to artist
